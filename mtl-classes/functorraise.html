FunctorRaise

FunctorRaise[F, E] expresses the ability to raise errors of type E in a functorial F[_] context. This means that a value of type F[A] may contain no A values but instead an E error value, and further map calls will not have any effect due to there being no value to execute the passed function on.

In that sense it is a less powerful version of ApplicativeError found in cats-core. In general, however, raiseError from ApplicativeError and raise from FunctorRaise should be equivalent.

The raise function has the following signature:

trait FunctorRaise[F[_], E] {
  def raise[A](e: E): F[A]
}

import cats._
// import cats._

import cats.data._
// import cats.data._

import cats.implicits._
// import cats.implicits._

import cats.mtl._
// import cats.mtl._

import cats.mtl.implicits._
// import cats.mtl.implicits._

def parseNumber[F[_]: Applicative](in: String)(implicit F: FunctorRaise[F, String]): F[Int] = {
  if (in.matches("-?[0-9]+")) in.toInt.pure[F]
  else F.raise(show"'$in' could not be parsed as a number")
}
// parseNumber: [F[_]](in: String)(implicit evidence$1: cats.Applicative[F], implicit F: cats.mtl.FunctorRaise[F,String])F[Int]

val valid = parseNumber[Either[String, ?]]("123")
// valid: scala.util.Either[String,Int] = Right(123)

val invalid = parseNumber[Either[String, ?]]("123abc")
// invalid: scala.util.Either[String,Int] = Left('123abc' could not be parsed as a number)
